# Пример интеграции платежной формы СБП с корректной обработкой deeplink в Android WebView

Данный пример использует действующую страницу оплаты Райффайзен банка в пользу Благотворительново фонда Хабенского, представлено два примера:
* реализация платежной формы банка, где уже поддержан виджет выбора банка - https://pay.raif.ru/pay/demo.html
* прямая реализация API виджета НСПК (без кэширования списка) - https://widget.cbrpay.ru/v1

Логика обработки переходов общая, так как часть банков возвращают веб-страницу, а не готовый deeplink.

---

## 🚀 Ключевые требования для корректной работы deeplink в WebView

1. **Настройки WebView**
    - `javaScriptEnabled = true`
    - `setSupportMultipleWindows = false` — отключает создание всплывающих окон, все ссылки открываются в основном WebView.

2. **Кастомный WebViewClient**  
   Переопределите метод `shouldOverrideUrlLoading`.

3. **Обработка ссылок**
    - Для всех ссылок, **не начинающихся с `http://` или `https://`** (например, `bank100000000007://...`):
        * Создайте `Intent(Intent.ACTION_VIEW, Uri.parse(url))`.
        * Запустите его через `startActivity` в блоке `try-catch`.
        * Верните `true`, чтобы WebView **не пытался загрузить этот URL** как страницу.
    - Для обычных `http://` или `https://` ссылок верните `false`, разрешая WebView загрузить их.

4. **Обязательно добавьте разрешение `INTERNET` в манифест**
   ```xml
   <uses-permission android:name="android.permission.INTERNET" />

## 🧪 Почему такой подход?
Использование startActivity без предварительной проверки resolveActivity (или с обработкой исключения) гарантирует, что deeplink будет передан системе Android. Даже если проверка resolveActivity по какой-то причине не находит установленное приложение (например, из-за кэширования или особенностей intent-фильтров), вызов startActivity либо успешно запустит приложение, либо выбросит исключение, которое можно обработать (показать пользователю сообщение или загрузить fallback-страницу).

Таким образом, мы избегаем ситуации, когда WebView пытается загрузить deeplink как обычный URL и показывает ошибку.
